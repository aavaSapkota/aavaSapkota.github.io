Arrays and Arraylists

In this tutorial, you will be learning about how to declare, initialize and use arrays and arraylists. If you aren’t familiar with any of the following prerequisites, I highly recommend checking them out for a better understanding of the content listed here.

Prerequisites:

  • Primitive data types
  • Strings
  • Variables
  • For Loops
  • For-Each Loops

ARRAYS

An array is a data structure with a fixed length that stores values of the same data type. Arrays can hold any data type, including other classes, and variable values. You can think of an array almost like a work book. It has a fixed number of pages, and content can be added, taken away or changed, and just like a workbook, an array is treated like an object. Throughout the tutorial, you will see the different ways to declare, initialize and manipulate arrays.

Declaring arrays:

We declare arrays to let the program know that we will be needing space for this object immediately or later in the program. Here are a few ways you can create an array:

  1. Data_type [] array_name = new Data_type[length_of_array];
  2. Data_type [] array_name = {elements, separated, by, commas};
  3. Data_type [] array_name;

Let's look at the first declaration option. In the first declaration, the array is declared and a new instance of the array is created. The key to this is the new keyword. Whenever included in a declaration, it instantiates whatever object it is being created, and stores a location in memory for them. In the first declaration, not only is a new array created in the program, and stored in memory, but it’s capacity is also initialized. This means that the array can not hold more than lenth_of_array elements (NOTE: each value stored in an array is referred to as an element).

The next declaration has the same beginning as the first, but after the equal sign, there are curly brackets containing the array’s elements. This declaration is used to both declare and initialize the array all on one line. This style is used when the programmer (you) knows all of the elements that will be stored in the array. (NOTE: when using this style, you can only use the curly brackets to store all of your elements, only when you declare the array. You cannot declare the array and then later on initialize the array with curly brackets). Unlike the previous declaration, the instance of the array is automatically created, and the keyword new does not need to be explicitly declared.

Finally, in the last declaration, you will see that no new instance of the array is created. This method is a good idea if you want to declare the array so it affects a specific scope (like a global variable), but don’t know the size or elements to be included in the array. This method is more to just declare the identifier for the array to be used later in the program.

Accessing and Initializing arrays:

Now that we have created an array, we still have to find a way to put values in it. To put values in an array, we first need to access it’s indices. An index is a location in the array, labeled by a number between 0-array_length-1. Exactly like String indexes, the first spot in the array is labeled 0, and increased until it is just one less than the declared length. To access a position in the array the following code is used:

arrayName[index]

When an array is first declared, it is automatically filled with default values.

Data Type: String/Objects int boolean char
Default Value null 0 false '\u0000'

To change or initialize an array the following code is used:

array[index]=newValue;

The new value must be the same data type as the array, but can be a piece of raw data (a string, integer, double, boolean, char etc) or it can be a variable of the same data type. One way to declare an array is declare each individual index, using the code above. However, a much faster and common way of declaring an array is with a loop. Let's initialize an array with random values between 1-10.

int [] numbers = new int[10];
for(int i=0; i<10){
    numbers[i]=(int)(Math.random()*10+1);
}

Notice how the for loop only goes through the values 0-9. If the for loop went started at 1 and ended at 10, the program would retrun an IndexOutOfBounds exception, because we would be trying to access index 10, when there is no index 10.

Example

Arrays are often used to store data that relates to one another. For example, if you had a classroom full of students, how would you store all of their names in a Java program? Many of you first time Java programmers may be thinking to make a bunch of variables that store each individual student’s name. Well, let's take a look at the code that would be required to make a classroom list of 10 students and display it:

public class Classroom{
    public static void main (String args[]){
        String student1 = “Samantha”;
        String student2 = “Sarah”;
        String student3 = “Amaya”;
        String student4 = “Ronald”;
        String student5 = “Gary”;
        String student6 = “Justin”;
        String student7 = “Celeste”;
        String student8 = “Eric”;
        String student9 = “Aava”;
        String student10 = “Steven”;

        System.out.println(student1);
        System.out.println(student2);
        System.out.println(student3);
        System.out.println(student4);
        System.out.println(student5);
        System.out.println(student6);
        System.out.println(student7);
        System.out.println(student8);
        System.out.println(student9);
        System.out.println(student10);
    }
}

As you can see, that takes up a lot of code. Instead lets apply what we just learned and make an array to store all of the names of the students. First lets decide what type of declaration we should do. For this situation, it would be best to use the second option because we already know all the values that will have to be stored in the array.

public class Classroom{

    public static void main (String args[]){
        String [] studentNames = {“Samantha”, “Sarah”, “Amaya”, “Ronald”, “Gary”, “Justin”, “Celeste”, “Eric”, “Aava”, “Steven”};
    }

}

The previous block of code required 10 lines to store 10 names. However, with an array, we were able to store 10 names in just 1 line of code. Next lets display the names. To do this, lets use a loop instead of printing out each index individually;

public class Classroom{

    public static void main (String args[]){
        String [] studentNames = {“Samantha”, “Sarah”, “Amaya”, “Ronald”, “Gary”, “Justin”, “Celeste”, “Eric”, “Aava”, “Steven”};

        for(int i=0;i<studentNames.length; i++){
                System.out.println(studentNames[i]);
        }
    }
}

As you can see, this code is much shorter than our original code, and gets the same job done. If you look closely at this code, you will see that in the for loop, the ending condition is

i<studentNames.length

. Just like how you can use .length() to get the length String variable, the same can be done with arrays. However, instead of using a method (.length()), arrays use a field (variable) to get their length. This means that instead of going through your program trying to remember the length of your array, you can simply call array.length and it will reveal the number of elements being stored in your array.

ARRAYLISTS

Arraylists are data structures that store class objects in a dynamic list. Unlike arrays, they can store as many objects/elements as they need, and like arrays, arraylists can only store one type of object. Going back to the work book analogy, if an array is a workbook with a set number of pages, then an Array List is a Binder with loose leaf paper pages. In the workbook, you will eventually come to the end, but with a binder, you can just continuously add in new pages.

Arraylists are objects and are related to the Collections class.To use an arraylist the following package will need to be imported:

Import java.util.*;

Declaring Arraylists

As mentioned above, Arraylists are object, so when declaring them, they are declared as Arraylist objects, with the type of data they hold declared in the arrow brackets (<>). The data type in the brackets cannot be any primitive data types (int, char, double, boolean). Just like arrays, Array Lists can be declared in different ways:

  1. ArrayList<Object_Class> arrayListName = new ArrayList<Object_Class>();
  2. ArrayList arrayListName;

As you can see, these two declarations look fairly similar to the array declarations above. However, since array lists don’t have a specified size, no size needs to be specified when declaring the arraylist. If you do want to specify the size of an array list, it can be done by placing the size in the curved brackets in the first declaration option. However, this is almost never done, as it remove the dynamic component of the arraylist.

Wrapper classes:

As mentioned above, primitive data types cannot be put in Arraylists, however it is still possible to store the numeric, character and boolean values that primitive datatypes store. Instead of using primitive datatypes, we use Wrapper classes. These wrapper classes are used to pass in primitive data type values through classes. Each primitive datatype has a class (Strings don’t need a wrapper class because a String is not a primitive data type).

Primitive Data Type int double char boolean
Wrapper class Integer Double Character Boolean

These wrapper classes however, are only required in the declaration of the arraylist. When using your arraylist in your program, you can add on primitive datatypes without having to make new wrapper class objects in your code. When adding primitive objects to arraylists, the arraylist automatically wraps the new piece of data without the programmer having to wrap the data manually.

Accessing and Initializing Arraylists

Unlike Arrays, there is only one way to initialize an Arraylist:

arraylist=new ArrayList<Object_Class>();

Once an arraylist is initialized, all of it’s values are set to null (because they are all objects, rather than primitive data types. Arraylists are similar to Strings in the sense that they both require methods to access their values. Here are a list of Array list methods and their uses:

arraylist.get(index); // <- returns the object at the specified index arraylist.set(index, newValue); // <- sets the object at the index with the new object arraylist.size(); // <- returns the size of the list arraylist.remove(index); // <- removes the object at index arraylist.add(newValue); // <- adds newValue to the end of the list arraylist.add(index, newValue);// <- inserts newValue at index.

Like arrays, arraylists can have their values modified, however, arraylists are more secure because they use methods to access the objects. This means that any data that is supposed to stay private can.

Example

Lets continue on with the previous example of the classroom. Throughout the year, new students will be arriving, but if a new student comes in, we won’t be able to add their name to the class list without making a whole new array. Instead of doing this, lets make the class list into an array list. This way, we can add in any new students that may come into the class.

import java.util.*;
public class Classroom{

    public static void main (String args[]){
        Arraylist<String> classList = new ArrayList<String>();

        classList.add(“Samantha”);
        classList.add(“Sarah”);
        classList.add(“Amaya”);
        classList.add(“Ronald”);
        classList.add(“Gary”);
        classList.add(“Justin”);
        classList.add(“Celeste”);
        classList.add(“Eric”);
        classList.add(“Aava”);
        classList.add(“Steven”);

        for(int i=0; i<classList.size(); i++){
            System.out.println(classList.get(i));
        }

    } }

While this code is longer, in the long run, it is much more secure and it will be easier to make modifications to the list rather than if it was an array.
