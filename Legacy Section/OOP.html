Object Oriented Programing

Object Oriented Programing is the Programming style where program flow is oriented around instances of classes. This is programing style is kwon to be used by many higher level languages such as java, javascript, python and c++. The concept behind object oriented program is surrounded by the idea of treating instances of classes as objects and trying to simulate real life behaviors through code. Object Oriented Programing can be used to simulate both literal and abstract object. Lets dig in a bit deeper.

Prerequisites:

  • Variables
  • Overloaded Methods

Classes and Objects

In java, nothing can run if it is not located within a class. Classes are the blueprints that create the objects we know today. They provide abstract fields that can be modified per object to create more realistic representations of objects. To create an object class, you need to first know the different aspects of the object that you are trying to create. For example if I wanted to create a Monster class, I would have to think about what kinds of things make up a monster. We can describe the characteristics of the monster as attribute and the actions that change the attributes behaviors. Translating all of this into more technical terminology, the attributes can be represented by variables (also referred to as fields) and the behaviors that change the attributes can be represented with methods.

Now that we have a basic idea as to what a class is, lets try to make one! Lets start with our attributes.

Attributes of a Monster:
    int eyes;
     int legs;
    int arms;
    String name;
    double energyLevel;

Behaviors of a Monster:
    -Eat
    -run

In the list above we have highlighted the different attributes and behaviors that monsters have that we want to represent in our program.

public class Monster{

    private int eyes;
    private int legs;
    private int arms;
    private String name;
    private double energyLevel;

    public Monster(){
        eyes = 2;
        legs = 2;
        arms = 2;
        name = “Ms.Monster”;
        energyLevel = 1000;
    }

    public Monster(int e, int l, int a, String n){
        eyes = e;
        legs = l;
        arms = a;
        name = n;
        energyLevel = 1000;
    }
}

Here in my class I have a few attributes of the monster declared globally. There are also two constructors added. The purpose of the constructor is to assign values to all of the attributes of the monster object. As you can see in the no-argument constructor, each of the attributes is given a value. This means that once an object is created, it will have 2 eyes, 2 legs, 2 arms, will be named Ms.Monster, and have a starting energy level of 1000. The other constructor has arguments which allows the object to be customized. The constructor that is called is dependent on what parameters are inserted when an object is instantiated.

Lets expand our class more and add methods. Methods are used to access the attributes and simulate behavior of an object. Typically there are a minimum of two methods relating to an attribute. These methods are called accessors and mutators (getters and setters). The accessor methods, get the values of the attributes and return them. The mutator methods, change the value of the attributes. Lets add some accessors and mutator methods into the class:

public class Monster{

    private int eyes;
    private int legs;
    private int arms;

Did you notice something? Not all the attributes have both get and set methods. All of the attributes have get methods, however energyLevel does not have a set method. This illustrates the security benefits of making instance variables private and using get and set methods to access the variables. The lack of a set method for the energy level attribute prevents direct tampering with the variable.

Lets move on to the behavior methods. Each of these methods are designed to also change meant to change the attributes, or other changes to the program. Lets add in some behavioural methods:

public class Monster{

    private int eyes;
    private int legs;
    private int arms;
    private String name;
    private double energyLevel;

    public Monster(){
        eyes = 2;
        legs = 2;
        arms = 2;
        name = “Ms.Monster”;
        energyLevel = 1000;
    }

    public void eat(){
        energyLevel+=10;
    }

    public void run(){
        energyLevel-=50;
    }

    public void growl(){
        System.out.println(“Grrrr…”);
    }

    public int getEyes(){
        return eyes;
    }

    public void setEyes(int e){
        eyes = e;
    }

    public int getLegs(){
        return legs;
    }

    public void setLegs(int l){
        legs = l;
    }

    public int getArms(){
        return arms;
    }

    public void setArms(int a){
        arms = a;
    }

    public int getName(){
        return name;
    }

    public void setName(int n){
        name = n;
    }

    public int getEnergyLevel(){
        return energyLevel;
    }

    public String toString(){
        return “Name: ” +name+“ Eyes: ” +eyes+“ Legs: ” +legs+ “ Energy Level: ” +energyLevel;
    }

}

The last method in this class is called a toString method. It is used to output a String that has all of the states of the instance variables. The toString method automatically is called when an object is printed. Without overloading the toString constructor, the object’s location in memory would be printed when the object is printed.

As shown in the code above, the behavioral methods have been added, and now, we have a full class! Now we can make all sorts of monsters! To do this, we need a class with a main method. This class is called the driver class and it controls the whole program.

public class MonsterTester{

In this tester class, two objects are instantiated, and the behavioural methods are called, all found inside the main method. Each program can only have one main method that runs the whole program. The output of this program is the following:

Inheritance:

In the section above, we made a Monsters class to tailor to Monsters. However, our class was really mostly directed towards land monsters, with their legs and arms. We need to expand our Universe and make more monsters, but we don’t want to completely start over, because many sea monsters have similar attributes as land monsters. Instead we are going to use the idea of inheritance to create a sea monsters’ class.

The idea of inheritance is based on the idea of super and sub classes. In this case, the monster class would be considered the superclass and the sea monsters class will be seen as the subclass. The subclass will inherit all the public methods from the super class, so this means we don’t have to rewrite all of those get and set methods! The relationship between the super and sub class is called a IS-A relationship, because a sea monster IS-A monster. Lets start building the sea monsters class. What are some things that only sea monsters have?

Polymorphism

The final step to our tutorial on OOP programing is the concept of polymorphism. Polymorphism is the concept of treating objects that are different the same. In the concept on OOP programing, it is when a subclass duplicates a method from the super class, but changes it’s content. That means, that when that method is called with an object from the subclass, the method in the subclass will override the superclass duplicated method. Lets see how this affects our monsters.

In the monster class, there is a method called growl that prints the statements: “Grrrr…”. While this would be accurate for Land Monsters, when Sea monsters growl, it sounds more like “griss…”. Lets go back to our Sea Monsters class.

public class SeaMonsters extends Monsters{
    private int gills;
    private int fins;
    public SeaMonsters(){
        super();
        gills = 2;
        fins = 4;
    }

    public int getGills(){
        return gills;
    }

    public void setGills(int g){
        gills = g;
    }

    public int getFins(){
        return fins;
    }

    public void setFins(int f){
        fins = f;
    }

    public void swim(){
        run();
    }

    public void growl(){
        System.out.println(“Grisss…”);
    }

    public String toString(){
         return “Name: ” +getName() +“ Eyes: ” +getEyes()+“ Fins: ” +fins+ “ Gills: ” +gills+ “ Energy Level: ” +energyLevel;
    }

}

Here, you can see the new growl method is added. This means that when we call it with a SeaMonsters object, it will print out “Grisss…” instead of “Grrrr…”.
